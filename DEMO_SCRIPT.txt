[1:00 — ARCHITECTURE]
Cue: (show assets/architecture_diagram.png)
(pause)
Let's walk through our system architecture flow node by node.
First, the user input enters the workflow graph and is intercepted by the Security Checkpoint node.
The Security Checkpoint scrubs PII and checks for prompt injections, routing safe queries to our central Orchestrator agent.
The Orchestrator agent acts as the brain, routing specific queries to either the Expense Tracker sub-agent or the Savings Advisor sub-agent.
Both sub-agents are wired with an MCP toolset that accesses the database tools to pull and push financial data.
Finally, any state-changing operations trigger the Human-in-the-Loop approval gate, prompting the user for explicit confirmation before completion.

[3:00 — SECURITY & MCP]
Our custom security checkpoint ensures complete compliance by redacting credit card/IBAN data, blocking prompt injections, preventing large transactions over $10,000, and writing compliance logs to stdout in JSON format.
Our local Model Context Protocol server exposes 5 key database tools: get_balance, get_transactions, add_transaction, get_savings_goals, and update_savings_goal, allowing seamless integration with our agents.
